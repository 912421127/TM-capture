# 商品排行模式切换 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在侧边栏新增可切换“实时 / 7天 / 30天”的商品排行表格，展示截图中的完整商品指标。

**Architecture:** 商品排行作为独立 feature 注册到公共请求捕获层。页面主世界通过统一的 `CAPTURE_TRIGGER` 接收模式参数并在页面上下文发起对应请求；后台仍按 `item-rank` 保存响应记录；侧边栏按当前模式触发请求并解析最近一条响应。解析和请求 URL 构造放在 feature 文件中，Vue 组件只负责交互和展示。

**Tech Stack:** Vue 3、TypeScript、WXT 内容脚本、Vitest。

## Global Constraints

- 三种模式使用当前页面已有的日期、token、关键词和分类筛选；插件暂不新增筛选器、分页器或排序器。
- 请求在生意参谋页面上下文中发起，复用登录状态、token 和当前页面的筛选上下文，不打开 DevTools、不使用 `chrome.debugger`、不改变页面位置。
- 完整展示截图中的 19 个指标，第一列固定并支持横向滚动。
- 7天/30天使用接口的对比字段；实时模式没有对比值时显示 `-`。
- 新增或修改核心逻辑写中文注释；UI 文案、错误提示和导出文案使用中文。
- 不新增依赖；修改 TypeScript/Vue 后运行 `npm run compile`，并运行 `npm test`。

 'itemStatus', 'itemCartCnt', 'itemCltByrCnt', 'visitCltRate', 'itmUv',

Assert `syncCrc` is preferred over `cycleCrc` for comparison display, a zero-only comparison becomes `null`/`-` to match the screenshot, and realtime rows with no comparison produce `change: null`. Assert malformed JSON, missing `data`, and missing metric values do not throw.
