# 商品排行实时与周期切换设计

## 概要

新增独立的“商品排行”接口模块，在插件侧边栏中支持“实时 / 7天 / 30天”切换，并按照生意参谋商品排行页面的横向表格样式展示完整商品指标。现有“数据概览”和“辅助抓包”功能保持不变。

## 接口与数据流

- 实时模式请求 `GET /cc/item/live/view/top.json`。
- 7天模式请求 `GET /cc/item/view/top.json`，使用 `dateType=recent7` 和当前页面对应的 7 天日期范围。
- 30天模式请求 `GET /cc/item/view/top.json`，使用 `dateType=recent30` 和当前页面对应的 30 天日期范围。
- 请求在生意参谋页面上下文中发起，复用登录状态、token 和当前页面的筛选上下文，不打开 DevTools、不使用 `chrome.debugger`、不改变页面位置。
- 后台按标签页和商品排行接口 ID 保存返回记录；切换模式后展示该模式最近一次成功响应。
- 返回数据从 `data.data` 解析商品行；接口错误、JSON 无效或数据为空时展示中文提示，不影响页面和其他采集功能。
- 商品排行主动请求会读取服务端 `recordCount`，连续请求所有服务端分页并合并响应；侧边栏只做本地分页，每页 10 条，保证所有商品都能查看。

## 展示与交互

- 侧边栏增加“商品排行”面板，顶部提供“实时 / 7天 / 30天”模式按钮和刷新按钮。
- 表格第一列固定，支持横向滚动；商品列展示图片、名称和商品 ID。
- 完整展示截图中的指标：支付金额、成功退款金额、支付件数、支付买家数、支付转化率、商品状态、商品加购件数、商品收藏人数、访问收藏转化率、商品访客数、商品浏览量、平均停留时长、商品详情页跳出率、搜索引导访客数、搜索引导支付买家数、搜索引导支付转化率、访客平均价值、推广消耗、推广直投 ROI。
- 每个指标单元格上下显示当前值和周期变化；7天/30天使用接口的 `cycleCrc`，实时模式没有对比值时显示 `-`。
- 历史模式默认按支付金额排序，实时模式沿用已抓到的页面默认按访客数排序；不新增排序控件。
- 页面窄屏下保留横向滚动、固定商品列和清晰的红绿变化色；无响应时显示“暂无数据”或具体错误提示。
- 分页栏显示总数、当前范围、上一页、页码和下一页；切换模式、刷新或新数据到达时回到第 1 页。

## 测试计划

- 测试三种模式的 URL、日期范围、排序和指标参数构造。
- 测试响应解析、商品信息、19 个指标、数值格式化和 `cycleCrc` 变化展示。
- 测试实时无对比值、空响应、错误响应和缺失字段的降级显示。
- 测试多页响应合并、商品 ID 去重、最后一页不足 10 条，以及本地分页范围计算。
- 运行 `npm run compile` 和 `npm test`；必要时执行生产构建确认 WXT 入口和 Vue 组件可打包。

## 假设与边界

- 三种模式使用当前页面已有的日期、token、关键词和分类筛选；插件暂不新增筛选器、分页器或排序器。
- 商品图片使用接口返回的 `pictUrl`，协议相对地址补为 `https:`。
- 完整请求头仍受浏览器页面 API 可见范围限制；功能只依赖页面脚本可读的请求参数和响应，不使用调试协议。
- 记录继续遵循当前插件的后台内存生命周期，刷新页面后重新采集。
